About the project

Built for Real Linux Engineers

linux-admin is an open-source Claude Code plugin that gives AI a senior Linux administrator mental model — evidence-first, safety-gated, distro-aware.

Mission

Why This Exists

linux-admin exists because AI assistants give generic Linux advice that ignores distro differences, safety constraints, and evidence-first diagnostics. This plugin gives Claude Code the structured mental model of a senior SRE: observe before acting, gather evidence before recommending, confirm before executing anything disruptive.

Audience

Who It Helps

linux-admin is designed for anyone who operates Linux systems and needs an AI that reasons correctly about what it's doing.

👨‍💻
Linux Admins
Managing production servers and diagnosing failures under pressure. Get expert guidance without leaving your terminal.
🔧
SREs / DevOps
On-call triage, incident response, and fleet operations. Structured diagnostics that match an experienced SRE's mental model.
🏗️
Platform Teams
Infrastructure migrations, kernel tuning, and container workloads. Distro-aware guidance across the whole stack.
🎓
Students
Learning Linux troubleshooting with expert guidance. Understand why a command is chosen, not just what it is.
🚀
Startups
Teams without a dedicated Linux admin. Get production-grade SRE reasoning without the headcount cost.
🏢
Enterprises
Augmenting existing ops workflows with AI assistance. Safety-gated by design — no rogue destructive commands.

Contributing

linux-admin is open to contributions. New skills, documentation improvements, and test additions are all welcome.

Documentation

🧪
Tests
Add test scenarios under tests/. Test cases should cover the diagnostic method, not just the final answer.
