What it does

SRE intelligence, built into Claude Code

Not a chatbot. A structured mental model — the way a seasoned Linux administrator thinks when something breaks at 2 AM.

🔍
Read-only-first diagnostics

Gathers evidence before touching anything. Follows the observe → hypothesize → verify → fix loop every senior SRE uses. No destructive action runs before you have a diagnosis.

🐧
Distro-aware command selection

Detects Ubuntu, RHEL, Fedora, SUSE, Arch. Uses the right package manager and service manager for each.

🛡️
Safety gates on destructive commands

PreToolUse hook classifies every command. Class 3 (rm -rf, dd, mkfs) is blocked. Class 2 requires explicit confirmation.

🎯
Evidence-based root-cause analysis

No guessing. Chases the error through logs, service dependencies, kernel messages, and config files.

⚡
58 task-specific skills

🤖
Multi-agent platform support

Claude Code native. Adapters for Codex (AGENTS.md) and Gemini (GEMINI.md) included. Works wherever your AI agent runs.

Command safety

Three-class protection model

Every shell command passes through a classification gate before execution. No exceptions.

Class R/O
Read-only — instant pass

Diagnostic commands run immediately with zero friction. Evidence collection has no gating.

cat · journalctl · systemctl status · ss · ip · df · du · ps · lsof · dmesg
Class 2
Disruptive — confirm required

Command halts. Warning is shown. User must type 'yes' to proceed. Never skipped.

systemctl stop · systemctl restart · firewall-cmd · sysctl -w · mount · iptables -A
Class 3
Destructive — blocked entirely

Plugin refuses, explains why, and suggests a safer alternative. Not a warning — a hard block.

rm -rf / · dd if=/dev/zero · mkfs.* · shred · wipefs · DROP TABLE · userdel -r
