Get started

Install in 30 seconds

Three ways to install the linux-admin plugin. Choose the method that fits your workflow. The CLI marketplace method is recommended for most users.

Methods

Three install methods

All three methods produce the same result — pick whichever fits your workflow.

CLI Marketplace
Recommended for most users
Recommended
terminal
# Add marketplace
$ claude plugin marketplace add rushikeshsakharleofficial/we-are-linux-administrators

# Install plugin
$ claude plugin install linux-admin@we-are-linux-administrators

Validate

Confirm the install

Run these two checks after installation to confirm everything is wired up correctly.

1. Validate plugin JSON

Check that the plugin manifest is valid JSON. No output means success.

$ python3 -m json.tool .claude-plugin/plugin.json >/dev/null
# Should print nothing (valid JSON)

2. Reload plugins

Inside Claude Code, run /reload-plugins to pick up the install without restarting.

› /reload-plugins
# Inside Claude Code — reloads all plugins

If plugin.json validation fails, re-clone the repository — the file may have been corrupted during download. If /reload-plugins reports no skills found, verify that the plugin directory path is correct and that plugin.json exists at the root of the repository.

First steps

First commands to try

Paste any of these directly into Claude Code. Each exercises a different category of expertise.

› /linux-admin:diagnose nginx.service failed after config change
› /linux-admin:network can ping 8.8.8.8 but DNS fails
› /linux-admin:storage /var is full but du does not match df
› /linux-admin:kernel-expert triage kernel panic
› /linux-admin:migration-expert plan CentOS 7 to Rocky 9 migration

You can also use skills without a colon prefix in conversational form: just describe the problem and mention the skill name. The structured /linux-admin:skill-name form invokes the expert module directly.

Contributors

Local preview

Working on the plugin or this website? Serve the site locally to preview changes before pushing.

# Serve the site locally
$ cd site
$ python3 -m http.server 8080
# Open http://localhost:8080

The site uses only static HTML, CSS, and vanilla JS — no build step required. Any browser-served at http://localhost:8080 will load all assets correctly including fonts, theme toggle, and copy buttons.

Development

Reload after skill edits

When you edit a skill file in the repository, reload the plugin inside your Claude Code session to pick up the changes immediately.

› /reload-plugins
# Reloads all installed plugins from disk

No restart required. Claude Code hot-reloads plugin skills on demand. Changes to .md skill files in the skills/ directory take effect after the next /reload-plugins.

Compatibility

Distro compatibility

linux-admin detects your distribution automatically and adapts commands, package manager calls, and file paths accordingly.

Ubuntu 20.04 Ubuntu 22.04 Ubuntu 24.04 Debian 11 Debian 12 RHEL 8 RHEL 9 CentOS Stream Rocky Linux AlmaLinux Fedora 38+ SUSE / openSUSE Arch Linux Alpine Linux
🐧
Distro detection

Reads /etc/os-release to identify the distro and version, then selects the correct package manager and init system.

📦
Package manager aware

Switches between apt, dnf, rpm, zypper, and pacman automatically. Never uses the wrong package manager for your distro.

🗂️
Path aware

Knows that RHEL uses /etc/sysconfig/ and Debian uses /etc/default/. No manual path correction needed.

If you run on a distro not listed here, linux-admin falls back to POSIX-safe commands and flags when it cannot detect the distribution. Open an issue on GitHub if you encounter distro-specific problems — contributions adding new distro support are welcome.
