How it works

Security is more than one checklist

The skill follows a lifecycle model: identify assets, protect controls, detect abuse, respond safely, and verify recovery.

🧭

Role-aware audit

Classifies the server first: web, mail, DNS, database, monitoring, jump host, VPN/router, container host, backup, or low-memory VPS.

🛡️

Defensive validation

Uses safe, authorized checks to confirm whether controls block or detect expected weak paths. No stealth, no exploitation, no disruption.

📊

Security scoring

Scores attack surface, SSH, firewall, patching, sudo, systemd, sysctl, limits, MAC, logs, and backup recovery readiness.

Security domains

What security-expert reviews

Exposure

Ports, bind addresses, IPv4/IPv6 parity, firewall zones, NAT, cloud security groups, admin panels.

SSH/Auth

OpenSSH version, password/root login, PAM/MFA, fail2ban, auth logs, key ownership, emergency access.

Privilege

Sudoers, SUID/SGID, capabilities, service accounts, writable privileged paths, user/group drift.

Systemd

User/Group, NoNewPrivileges, ProtectSystem, ProtectHome, capabilities, resource limits, environment exposure.

Sysctl

Kernel exposure, ptrace/eBPF/perf restrictions, filesystem protections, redirect/source-route controls.

Limits

nofile, nproc, memlock, core dumps, TasksMax, MemoryMax, PAM limits, and blast-radius controls.

Detection

journald, rsyslog, auditd, fail2ban, file integrity, sudo/auth logs, retention, and time sync.

Recovery

Immutable/offline backups, restore tests, snapshots, backup credential separation, ransomware readiness.

Command

Use it from Claude Code

/linux-admin:security-expert audit this web server
/linux-admin:security-expert score this mail server security posture
/linux-admin:security-expert generate fix plan for failed SSH and sysctl checks
