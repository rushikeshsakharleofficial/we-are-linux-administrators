Privacy Policy

Security feedback is user-approved only.

linux-admin does not automatically upload server data, logs, configuration, findings, issues, or pull requests. Security feedback is prepared locally and submitted only after the user reviews and approves it.

No automatic submissions

The plugin must not create GitHub issues, pull requests, comments, reports, or feedback submissions without explicit user confirmation.

User approval required

Before any security finding is submitted, the user must see the exact title, body, metadata, labels, and target repository.

Sanitized feedback only

Users should remove private infrastructure details and share only the minimum safe evidence needed to improve the plugin.

How the feedback loop works

security-expert finds a gap
        ↓
generates a sanitized issue draft
        ↓
user reviews and approves
        ↓
GitHub issue is created
        ↓
maintainers improve skills, checklists, templates, and docs

This is a feedback loop for repository improvement. It is not automatic model training and does not use private server data without consent.

Repository policy

The same policy is documented in the GitHub repository in PRIVACY.md. If the website and repository ever differ, the stricter privacy rule should be followed.
