/linux-admin:diagnose
Main triage router. Routes any Linux problem to the right expert. The entry point for any unexpected failure.
Use when: any unexpected failure or symptom you're not sure how to classify.
/linux-admin:diagnose nginx.service keeps restarting
/linux-admin:boot
Boot failures, GRUB, initramfs, fstab issues. Full coverage of emergency-mode and recovery scenarios.
Use when: system won't boot or drops to emergency mode.
/linux-admin:boot kernel panic after kernel upgrade
/linux-admin:service
systemd service debugging. Restart loops, crash analysis, unit file issues, dependency chains.
Use when: a service crashes or enters a restart loop.
/linux-admin:service postgresql restart loop
/linux-admin:logs
journald, rsyslog, incident timeline construction. Correlates events across log sources to build a complete picture.

/linux-admin:logs build incident timeline for last 2 hours

Shell scripts, Ansible playbooks, and fleet-wide triage. Debugs why automation fails on specific nodes.

/linux-admin:automation check why deploy script fails on 3 nodes
/linux-admin:performance
CPU, memory, OOM, load average, I/O latency. Identifies bottlenecks using read-only diagnostic tools first.
Use when: high load average, slow response, or mysterious memory growth.
/linux-admin:performance high load average but CPU looks idle
/linux-admin:containers

/linux-admin:kernel-expert triage kernel panic after update
