/linux-admin:diagnose
Main triage router. Routes any Linux problem to the right expert. The entry point for any unexpected failure.
Use when: any unexpected failure or symptom you're not sure how to classify.
/linux-admin:diagnose nginx.service keeps restarting
/linux-admin:boot
Boot failures, GRUB, initramfs, fstab issues. Full coverage of emergency-mode and recovery scenarios.
Use when: system won't boot or drops to emergency mode.
/linux-admin:boot kernel panic after kernel upgrade
/linux-admin:service
systemd service debugging. Restart loops, crash analysis, unit file issues, dependency chains.
Use when: a service crashes or enters a restart loop.
/linux-admin:service postgresql restart loop
/linux-admin:logs
journald, rsyslog, incident timeline construction. Correlates events across log sources to build a complete picture.
Use when: investigating what happened in the past N hours.
/linux-admin:logs build incident timeline for last 2 hours
/linux-admin:automation
Shell scripts, Ansible playbooks, and fleet-wide triage. Debugs why automation fails on specific nodes.
Use when: deploy scripts fail on some nodes but not others.
/linux-admin:automation check why deploy script fails on 3 nodes
/linux-admin:performance
CPU, memory, OOM, load average, I/O latency. Identifies bottlenecks using read-only diagnostic tools first.
Use when: high load average, slow response, or mysterious memory growth.
/linux-admin:performance high load average but CPU looks idle
/linux-admin:containers
Docker, Podman, rootless containers. Networking failures, mount problems, exit-code analysis.
Use when: container crashes on startup or networking fails.
/linux-admin:containers app container exits immediately
/linux-admin:systemd-expert
Unit design, restart loops, cgroups, drop-ins, and hardening. The deep-dive systemd specialist.
Use when: a service won't stay up or you need to design a new unit file correctly.
/linux-admin:systemd-expert debug nginx restart loop
/linux-admin:sysctl-expert
Kernel runtime parameters, safe sysctl tuning. Reads current values, explains semantics, validates before writing.
Use when: tuning kernel networking, memory, or I/O parameters.
/linux-admin:sysctl-expert tune net.core.somaxconn safely
/linux-admin:limits-expert
ulimit, PAM limits, systemd Limit* directives. Traces the full resolution chain: shell → PAM → systemd.
Use when: too many open files, nproc exhaustion, or ulimit not sticking.
/linux-admin:limits-expert postgres too many open files
/linux-admin:kernel-expert
Kernel panic triage, kdump readiness, call trace reading. Interprets Oops messages and lockup signatures.
Use when: kernel panics, soft lockups, or RCU stalls.
/linux-admin:kernel-expert triage kernel panic after update
/linux-admin:date-timectl-expert
Timezone and NTP fixes via timedatectl. Handles wrong-time-after-migration and DST confusion.
Use when: system clock shows wrong time or timezone is incorrect after migration.
/linux-admin:date-timectl-expert timezone wrong after migration
/linux-admin:chrony-expert

Use when: clock drifts noticeably or NTP sources are unreliable.
/linux-admin:chrony-expert clock drifting 200ms per hour

/linux-admin:kernel panic after 6.8 upgrade
