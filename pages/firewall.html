Overview

Firewall Expertise

Full-stack firewall management across every major Linux netfilter frontend. From raw nftables rulesets to high-level UFW policies and large-scale ipset country blocks — with a lockout-safe approach hardwired into every operation.

🛡️
nftables
Modern netfilter frontend. Atomic rule replacement, sets, maps, and verdict chains in a single consistent syntax. Preferred for all new configurations on kernel 3.13+.
🔥
firewalld
Zone-based dynamic firewall daemon. Default on RHEL, Fedora, and Rocky Linux. Manages runtime and permanent rules separately — no service restart needed for live changes.
⛓️
iptables
Legacy netfilter interface. Supported for existing configurations and distros that predate nftables. New rulesets are written in nftables whenever possible.
🧱
UFW
Uncomplicated Firewall — the Ubuntu/Debian standard for simple port management. Ideal for single-server setups and beginners. Wraps iptables with a friendly CLI.
📋
ipset
Kernel-level IP set management for matching thousands of addresses efficiently. Used for country blocks, known bad-actor lists, and allowlists that would be impractical in plain iptables rules.
⇄
NAT & Masquerade
SNAT, DNAT, port forwarding, and masquerade rules for routers, VPN gateways, and container networking. Correct postrouting chain placement verified before application.
Safety Protocol

Lockout-Safe Protocol

Locking yourself out of a remote server with a bad firewall rule is one of the most painful self-inflicted outages in Linux administration. This protocol makes it structurally impossible.

The rule: Before any deny-all or restrictive ruleset is applied, an explicit accept rule for the current session's source IP is inserted at the top of the input chain. This rule is the last thing removed — only after connectivity is confirmed from a second terminal. Recovery rules use a timeout comment so they are easily identified and cleaned up.

01
Record Current IP
Capture the source address of the active SSH session using who -a, ss -tnp, or $SSH_CLIENT. Store it in a shell variable. Never trust what curl ifconfig.me returns — that is the NAT exit IP, not the session source.
02
Add Emergency Accept Rule
Insert an unconditional accept rule for the captured IP at position 1 of the input chain — above any deny rules. For nftables: nft insert rule inet filter input ip saddr $MY_IP accept. For iptables: iptables -I INPUT 1 -s $MY_IP -j ACCEPT.
03
Apply New Rules
Load the new ruleset atomically (nft -f /etc/nftables.conf) or apply changes incrementally. The emergency accept rule at the top of the chain guarantees the current session survives even if the new rules include a deny-all default policy.
04
Test from a Second Terminal
Open a new, independent SSH connection before removing the recovery rule. Confirm the new rules work as intended — services reachable, blocked ports actually blocked. Never close the original session first.
05
Remove Emergency Rule Only After Verified
Once connectivity is confirmed in the new terminal, delete the recovery rule by handle or by matching the comment. For nftables: nft delete rule inet filter input handle $(nft -a list ruleset | grep EMERGENCY | awk '{print $NF}'). The ruleset is now clean and intentional.

Tool Guide

Which Tool to Use

Choosing the right frontend matters — mixing tools on the same host leads to rule conflicts and unpredictable filtering behaviour.

Preferred
nftables
Modern replacement for iptables/ip6tables/arptables. Atomic flushes, native sets, single kernel subsystem. Use for all new Linux deployments on kernel 3.13+.
kernel 3.13+ atomic sets/maps
Zone-based
firewalld
RHEL/Fedora/Rocky default. Zones abstract interfaces and sources. Permanent + runtime layers prevent config drift after reboot. Integrates with NetworkManager.
RHEL/Fedora zones dynamic
Legacy
iptables
Avoid for new configurations. Maintain existing rulesets only. Non-atomic rule application creates race windows. Separate tools needed for IPv6, ARP, and bridge filtering.
legacy non-atomic avoid new
Simple
UFW
Ubuntu/Debian-friendly high-level interface. Hides netfilter complexity for single-server, port-based policies. Not suitable for complex multi-zone or NAT setups.
Ubuntu beginner simple
Performance
ipset
Kernel hash/bitmap sets for matching large address lists at O(1) cost. Use with iptables or nftables for country blocks and large blocklists. Complements, does not replace, nftables.
O(1) lookup large lists geo-block
Commands

Key Commands

The essential read-only inspection commands across all five tools. Safe to run on any live system before making changes.

# Dump entire nftables ruleset with handles
nft list ruleset
nft -a list ruleset # with handle numbers
# firewalld — active zones and rules
firewall-cmd --list-all
firewall-cmd --list-all-zones
firewall-cmd --get-active-zones
# Capture current session source IP
MY_IP=$(echo $SSH_CLIENT | awk '{print $1}')
# Verify: echo $MY_IP
# iptables — save complete ruleset
iptables-save # IPv4
ip6tables-save # IPv6
iptables -nvL --line-numbers
# UFW — verbose status with rule numbers
ufw status verbose
ufw status numbered
# ipset — list all sets with members
ipset list
ipset list <setname>
# Active listening sockets
ss -tlnp
