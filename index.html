What it does

SRE intelligence, built into Claude Code

Not a chatbot. A structured mental model — the way a seasoned Linux administrator thinks when something breaks at 2 AM.

🔍
Read-only-first diagnostics

Gathers evidence before touching anything. Follows the observe → hypothesize → verify → fix loop every senior SRE uses.

🐧
Distro-aware command selection

Detects Ubuntu/Debian, RHEL/CentOS, Fedora, SUSE, Arch. Uses the right package manager, service manager, and file paths for each.

🛡️
Safety gates on destructive commands

PreToolUse hook classifies every command. Class 3 (rm -rf, dd, mkfs) is blocked. Class 2 (systemctl stop, firewall changes) requires explicit confirmation.

🎯
Evidence-based root-cause analysis

No guessing. Chases the actual error back through logs, service dependencies, kernel messages, and config files to find the true root cause.

⚡
30+ task-specific skills

Deep-dive expert modules for sysctl, systemd, nftables, fail2ban, LVM, kdump, SSSD, PAM, ACLs, and more. Each is a focused specialist.

🤖
Multi-agent platform support

Claude Code native. Adapters for Codex (AGENTS.md) and Gemini (GEMINI.md) included. Works wherever your AI agent runs.

Safety policy

Three-class command safety gate

A PreToolUse hook inspects every command before it executes. Commands are classified by blast radius and handled accordingly — no surprises, no silent destruction.

Class R/O
Read-only

Diagnostic commands pass through instantly. Zero friction on gathering evidence.

cat, journalctl, ss, ip, df, du, systemctl status, ps, top, lsof, netstat
Class 2
Confirm required

Disruptive commands halt and display a clear warning. You confirm, then they run.

systemctl stop|restart, firewall-cmd --add-rule, iptables -A, sysctl -w
Class 3
Blocked outright

Destructive commands are denied entirely. The plugin explains why and suggests a safe alternative.

rm -rf /, dd if=/dev/zero, mkfs.*, shred, wipefs, drop database
